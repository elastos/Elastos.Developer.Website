    • Files: object
      • newDownloadExecutable: function

        Creates a new DownloadExecutable for setScript().

        filePath can be hardcoded (ex: "icons/userProfile.png"), or composed with values of call() parameters (ex: "icons/appIcon_${params.path}" + call() with a "path" field in params).

        Please note that using call() parameters can potentially be harmful if you make it too permissive, as a remote user could potentially access any file from your app storage. For example, we highly discourage registering "filePath:${params.path}" that would be very flexible from the application to download any file, but would also allow anyone pass an unexpected path and access other files. Also, it is suggested to use access conditions to verify that the caller is allowed to access a resoruce based on the target path.

          • (filePath: string, executableName?: string): DownloadExecutable

          • Parameters

            • filePath: string
            • Optional executableName: string

            Returns DownloadExecutable
